# 📱 모바일 반응형 웹 개선 계획서

## 1. 배경 및 목적

- **현황**: 현재 `st.columns(6)`를 사용하여 로또 번호를 표시하고 있어, 모바일 등 좁은 화면에서 레이아웃이 깨지거나 가독성이 떨어짐.
- **목표**: CSS Flexbox와 미디어 쿼리를 도입하여 기기 화면 크기에 관계없이 최적화된 레이아웃을 제공.

## 2. 주요 개선 사항

### 2.1. 로또 번호 표시 방식 변경

- **기존**: Streamlit의 `st.columns` 사용 (고정 레이아웃).
- **변경**: HTML/CSS `flex-wrap` 속성 사용.
  - 화면이 넓으면 한 줄에 표시.
  - 화면이 좁으면 자동으로 줄바꿈.

### 2.2. 모바일 전용 스타일 적용

- **미디어 쿼리 (`@media`)**: 화면 너비 600px 이하일 때 스타일 조정.
- **공 크기 축소**: 45px -> 35px.
- **폰트 크기 조정**: 가독성 확보.

### 2.3. UI 컴포넌트 최적화

- 사이드바 및 메인 콘텐츠 영역의 여백 조정.
- 버튼 및 입력 필드의 모바일 터치 편의성 고려.

## 3. 구현 상세

### 3.1. CSS 주입 함수 (`inject_custom_css`)

- `src/web_app.py`에 커스텀 CSS를 주입하는 함수 추가.
- `.lotto-ball-container`, `.lotto-ball` 클래스 정의.

### 3.2. 페이지별 적용

- `recommendation_page`: 추천 번호 표시 영역.
- `number_theme_page`: 코어/고정 번호 표시 영역.
- `data_update_page`: 당첨 번호 표시 영역.
- `display_retraining_ui`: 재학습 결과 표시 영역.

## 4. 예상 효과

- 모바일 기기에서의 사용자 경험(UX) 대폭 향상.
- 화면 크기에 구애받지 않는 일관된 디자인 제공.
